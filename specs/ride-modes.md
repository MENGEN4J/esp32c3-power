# v2.0 骑行模式规格

> Source of Truth：v2.0 骑行模式与数据源架构规划
> AI 修改骑行模式、功率引擎、Web 配置页时，先加载此文件确认目标行为

---

## 版本目标

v2.0 聚焦“骑行模式引擎”，先不实现 FIT 活动文件回放。设备仍保持 BLE 优先和 WiFi/BLE 互斥架构，通过 Web 配置页面选择骑行模式，保存后重启恢复 BLE，由主循环按所选模式生成功率、踏频和心率数据。

## 非目标

- 不在 v2.0.0 实现 FIT 文件上传、解析或活动回放。
- 不实现 WiFi 与 BLE 同时运行的实时网页遥控。
- 不新增 FTMS 智能骑行台控制服务。
- 不在 BLE 通知回调中创建大对象或解析复杂数据。

## 骑行模式

| 模式 ID | 名称 | 目标体验 | 功率行为 | 踏频行为 | 心率行为 |
|---------|------|----------|----------|----------|----------|
| `steady` | 固定功率模式 | 室内稳定踩踏 | 围绕用户表单功率小幅波动 | 围绕用户表单踏频小幅波动 | 围绕用户表单心率缓慢波动 |
| `road` | 真实路骑模式 | 模拟户外路骑起伏 | 使用内置状态机，支持滑行 0W、短时冲刺、高低功率交替 | 使用内置状态机联动 | 使用内置心率曲线延迟变化 |
| `interval` | 间歇训练模式 | 模拟高低强度训练 | 使用内置高强度/恢复功率区间 | 使用内置高强度/恢复踏频区间 | 使用内置训练心率曲线 |
| `random` | 随机巡航模式 | 轻量随机变化 | 使用内置 110-300W 随机游走 | 使用内置 65-94RPM 随机游走 | 根据内置功率趋势平滑变化 |

## 默认模式

- 新设备默认使用 `steady` 模式。
- 从 v1.x 升级到 v2.0 时，如果配置文件没有模式字段，按 `steady` 处理。
- 若配置文件损坏或模式未知，回退到 `steady`，并保留设备 BLE 可用。

## 数据源接口

v2.0 应将主循环改为读取统一快照，而不是分别读取 `power`、`cadence`、`heartrate` 属性。

| 接口 | 说明 |
|------|------|
| `get_snapshot(now_ms)` | 返回当前 BLE 应广播的数据快照 |
| `set_mode(mode)` | 设置骑行模式并标记配置待保存 |
| `get_mode()` | 返回当前模式 ID |
| `reset_mode_state()` | 模式切换后重置内部状态，避免旧模式状态污染新模式 |

快照字段：

| 字段 | 类型 | 范围 | 说明 |
|------|------|------|------|
| `power` | int | 0-2000 | 当前功率 |
| `cadence` | int | 20-120 | 当前踏频 |
| `heartrate` | int | 60-200 | 当前心率 |
| `mode` | str | 模式 ID | 当前骑行模式 |

## 固定功率模式

### 行为规则

- 功率围绕 `base_power` 小幅波动。
- 踏频围绕 `cadence_base` 小幅波动。
- 若用户设置心率，则心率围绕用户设定值稳定波动。
- 若用户未设置心率，则心率根据基准功率估算，并做慢速平滑。
- 只有固定功率模式允许通过 Web 表单或按钮修改功率、踏频、心率基础值。

### 建议参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 功率波动 | -5W 到 +5W | 稳定功率体验 |
| 踏频波动 | -2RPM 到 +2RPM | 稳定踏频体验 |
| 心率波动 | -1BPM 到 +1BPM | 稳定心率体验 |
| 心率平滑系数 | 0.95 | 防止心率跳变 |

### 验收标准

- GIVEN 模式为 `steady`
- WHEN 连续运行 5 分钟
- THEN 功率不应出现 0W 滑行或大幅冲刺
- THEN 踏频和心率变化应平滑

## 真实路骑模式

### 行为规则

真实路骑模式模拟户外骑行，包含滑行、巡航、爬坡、冲刺和恢复等状态。该模式不依赖 FIT 回放，而是使用轻量状态机生成数据。

参数依据：从 `ride-0-2026-05-20-05-31-24.fit` 提取 5956 条 record。真实样本功率中位数约 197W，P75 约 252W，P95 约 327W，380W 以上冲刺约占 1.1%，0W 或接近 0W 滑行约占 6.5%。因此真实路骑模式以巡航和爬坡为主，冲刺短且少，滑行时心率保持滞后下降。

| 状态 | 功率范围 | 踏频趋势 | 心率趋势 | 说明 |
|------|----------|----------|----------|------|
| `coast` | 0W 到 15W | 20-28RPM | 目标约 140BPM | 溜腿或短暂不踩，心率滞后下降 |
| `cruise` | 130W 到 220W | 66-90RPM | 目标约 158BPM | 平路巡航，占比最高 |
| `climb` | 235W 到 335W | 75-93RPM | 目标约 166BPM | 爬坡或逆风 |
| `surge` | 385W 到 480W | 78-100RPM | 目标约 170BPM | 短时发力，持续时间短 |
| `recover` | 55W 到 110W | 50-88RPM | 目标约 150BPM | 高强度后恢复 |

### 状态机

- 状态持续时间按状态区分：滑行 3-18 秒、巡航 5-35 秒、爬坡 4-28 秒、冲刺 1-8 秒、恢复 2-12 秒。
- 状态切换使用权重随机，不允许长时间停留在 `surge`。
- `surge` 之后优先进入 `recover`。
- `coast` 可以从 `cruise` 或 `recover` 进入。
- 心率不立即跟随功率跳变，应有延迟和惯性。
- 真实路骑模式不使用用户表单里的功率、踏频、心率数值。

### 建议状态权重

| 当前状态 | 可切换状态 | 权重说明 |
|----------|------------|----------|
| `cruise` | `coast` / `climb` / `surge` / `cruise` | 巡航概率最高 |
| `climb` | `recover` / `cruise` / `surge` | 爬坡后多恢复或巡航 |
| `surge` | `recover` / `cruise` | 冲刺后必须降低强度 |
| `recover` | `cruise` / `coast` / `climb` | 恢复后回到巡航 |
| `coast` | `cruise` / `recover` | 滑行后重新踩踏 |

### 验收标准

- GIVEN 模式为 `road`
- WHEN 连续运行 30 分钟
- THEN 应出现至少一次 0W 或接近 0W 滑行
- THEN 应出现多次高于基准功率 100W 以上的发力段
- THEN 心率应随高功率段延迟上升，不应每秒剧烈跳变
- THEN 踏频应随发力段提升，滑行段下降

## 间歇训练模式

### 行为规则

- 以固定周期在高强度段和恢复段之间切换。
- 高强度段功率高于基准功率。
- 恢复段功率低于基准功率。
- 心率在多个高强度段后逐渐累积上升，恢复段缓慢下降。

### 建议参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 高强度时长 | 60 秒 | 可在 Web 页面配置 |
| 恢复时长 | 120 秒 | 可在 Web 页面配置 |
| 高强度功率 | 285-355W | 参考真实样本 P90-P95 附近 |
| 恢复功率 | 85-145W | 参考真实样本恢复段 |
| 高强度踏频 | 84-96RPM | 参考真实样本高功率段 |
| 恢复踏频 | 62-84RPM | 参考真实样本低功率段 |
| 高强度目标心率 | 约 174BPM | 平滑上升 |
| 恢复目标心率 | 约 148BPM | 缓慢下降 |

间歇训练模式不使用用户表单里的功率、踏频、心率数值。

## 随机巡航模式

### 行为规则

- 使用随机游走生成中等幅度变化。
- 不出现真实路骑模式中的强状态语义。
- 适合作为比固定模式更自然但又不剧烈的模拟模式。
- 功率在 110-300W 内随机游走，覆盖真实样本 P10-P90 主体区间。
- 踏频在 65-94RPM 内随机游走，覆盖真实样本 P10-P95 主体区间。
- 心率约在 135-166BPM 之间随功率趋势平滑变化。
- 随机巡航模式不使用用户表单里的功率、踏频、心率数值。

## 持久化格式

`power_config.json` 在 v2.0 中继续使用短 key，新增模式字段：

```json
{"p":200,"c":90,"h":140,"m":"steady"}
```

| Key | 说明 |
|-----|------|
| `p` | 基准功率 |
| `c` | 基准踏频 |
| `h` | 用户心率或上次心率 |
| `m` | 骑行模式 ID |

## Web 配置

配置页应新增“骑行模式”区域：

- 固定功率模式：说明“使用下方表单数值，功率、踏频、心率稳定小幅波动”。
- 真实路骑模式：说明“内置滑行、巡航、爬坡、冲刺、恢复曲线，不使用表单数值”。
- 间歇训练模式：说明“内置 60 秒高强度 + 120 秒恢复，不使用表单数值”。
- 随机巡航模式：说明“内置 110-300W 随机游走，不使用表单数值”。

表单行为：

- 当模式为 `steady` 时，允许编辑功率、踏频、心率。
- 当模式不为 `steady` 时，前端禁用功率、踏频、心率输入框。
- 后端收到非 `steady` 模式提交时，必须忽略功率、踏频、心率字段，仅保存模式。
- 按钮短按/中按仅在 `steady` 模式下调整功率，其他模式下不修改配置。

保存模式后仍保持现有流程：

- Web 页面提交配置。
- 立即写入 `power_config.json`。
- 5 秒后关闭 WiFi 并重启。
- BLE 恢复后按新模式广播。

## 硬件约束

### WiFi/BLE 互斥

模式选择发生在 WiFi 配置阶段，BLE 已暂停。保存后设备重启恢复 BLE。v2.0 不实现网页实时调参。

### 内存约束

- 模式状态机不得在主循环中创建大对象。
- `get_snapshot(now_ms)` 返回可复用结构或短生命周期小对象。
- BLE 通知路径继续复用 `ble_service.py` 的预分配缓冲区。
- 不引入 CPython 专有库或复杂数学库。

## 迁移策略

- 读取旧版配置时，缺失 `m` 字段则使用 `steady`。
- 保存配置时写入 `m` 字段。
- 模式字段非法时回退 `steady` 并在日志中记录。
- 不改变 BLE 数据包格式，保证现有骑行 App 兼容。

## 测试要求

- 模拟模式兼容：默认配置下，行为接近 v1.x。
- 模式持久化：保存后重启仍保持用户选择。
- BLE 兼容：骑行 App 能正常读取功率、踏频、心率。
- 长稳测试：`road` 模式连续运行 30 分钟无 WDT 重启。
- 内存测试：运行期间 `gc.mem_free()` 不应持续下降。
- 范围测试：所有模式输出必须满足功率、踏频、心率范围限制。

## 后续版本预留

FIT 活动回放保留为 v2.1+ 候选能力。v2.0 的骑行模式状态机应保持与未来回放引擎兼容，即主循环只依赖统一快照接口，而不关心数据来自模拟状态机还是活动文件。
